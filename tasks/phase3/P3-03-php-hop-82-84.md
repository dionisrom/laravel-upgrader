## Context from PRD & TRD

### Docker Images (PRD §11.1)

| Image | PHP Base | Key Breaking Changes | Rector Set |
|---|---|---|---|
| `upgrader:php-8.2-to-8.3` | PHP 8.3 | Typed class constants, `#[Override]`, readonly amendments | `LevelSetList::UP_TO_PHP_83` |
| `upgrader:php-8.3-to-8.4` | PHP 8.4 | Property hooks, asymmetric visibility, implicit nullable deprecated | `LevelSetList::UP_TO_PHP_84` |

### PHP 8.4 Property Hooks Note

Property hooks are a major new feature. Rector handles the deprecation of implicit nullable parameters which is the breaking change, but property hooks themselves are opt-in. The hop should:
1. Fix implicit nullable parameters (`Type $x = null` → `?Type $x = null`)
2. Flag dynamic property usage (deprecated in 8.2, error in 8.4 without `#[AllowDynamicProperties]`)
3. NOT auto-convert existing code to use property hooks (that's a refactoring choice, not an upgrade requirement)

### Rector Config (PRD §11.4)

```php
// rector-configs/rector.php83-to-php84.php
return RectorConfig::configure()
    ->withPaths(['/workspace'])
    ->withSets([LevelSetList::UP_TO_PHP_84])
    ->withPhpSets(php84: true);
```
